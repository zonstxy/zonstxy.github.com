# 准备工作

在进行平台打包之前，首先需要确定一些基础的第三方sdk配置；首先需要按照以往的包名命名规则：com.xq5.<平台拼音>或者com.zonst.zzgame.<平台拼音>进行包名命名；有特殊需求可以作IOS/Android的包名区分。另外还需要确定平台的游戏名称和ICON，SDK申请时会用到。

# 提交申请

确定包名、平台名、游戏ICON之后，可以分别向各位负责SDK申请的大佬提交申请，负责人员如下：

微信、百度：周文兵

擦肩：蔡家昌

闲聊：余强

一般申请周期会有2-3天，在确定各平台SDK无误的情况下可以进行下一步

static NSString *SDK_WX_APPID = @"wx876c42c7bccb35e2";

static NSString* bugly_ID = @"9fa1b77262";

# 关于擦肩

擦肩的ios和android对应的APP_ID是一致的，值得注意的是除了以上的配置外，需要在项目的config.lua文件下加上如下内容，否则无法唤醒擦肩：

cajianAppKey_android="cjb182tvjzjpez"
cajianAppKey_ios="cjb182tvjzjpez"

# 关于Bugly

Bugly是腾讯出的用于异常上报和运营统计的第三方产品，在解决平台异常崩溃方面十分有效。ios工程下找到target->Build Phases->Run Script修改APP_ID和Key即可。Android的配置分两部分：在app/build.gradle中加入如下内容：

bugly {
    appId = '75b72a2ba0'
    appKey = 'e4e15634-652f-4880-aa59-edc4a20f7876'
}

然后在app/src/org.cocos2dx/cpp/AppActivity中替换已申请的APP_ID:

CrashReport.initCrashReport(getApplicationContext(), "你的APP_ID", false, strategy);
